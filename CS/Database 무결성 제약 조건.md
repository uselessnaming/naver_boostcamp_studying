# 데이터 베이스 무결성 제약 조건
무결성이란 데이터가 결함이 없다, 즉 데이터를 정확하고 유효하게 유지한다는 것이다.           
이 무결성 제약 조건에는 개체 무결성 제약 조건, 참조 무결성 제약 조건, 도메인 무결성 제약 조건 총 3가지가 존재한다. 

## 개체 무결성 제약 조건
기본 키로 지정된 값들은 중복 값이 없이 유일해야 하며, NULL일 수 없다는 조건이다.        
기본 키는 해당 튜플을 표현하는 고유한 값이기 때문에 중복되면 안되고, NULL로 처리할 수 없다.     

필드 설정 시 기본 키 필드에 `PRIMARY KEY`를 붙이게 된다.      
`PRIMARY KEY`를 처리한 후 필드를 살펴보면 NOT NULL 처리가 되어 있음을 알 수 있다.

## 참조 무결성 제약 조건
외래 키 값은 NULL 이거나 다른 테이블의 기본 키와 같아야 한다는 조건이다.        
즉, 각 릴레이션은 참조할 수 없는 외래키 값을 가질 수 없다는 것이다.

필드 제약 조건 설정 시 `CONSTRAINT [제약 조건 명] FOREIGN KEY [외래 키] REFERENCES [테이블명]([기본 키])` 와 같이 제약 조건을 추가한다.     
해당 테이블의 기본 키를 현재 테이블에서 기본 키로 사용하게 된다.
