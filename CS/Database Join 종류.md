# Database Join 종류
RDB에서는 여러 테이블의 기준을 정하고 기준을 통해 테이블을 합친 후에 데이터를 관리할 수 있습니다.       
이런 연산을 JOIN 연산이라 합니다.       

## FULL Join
양 쪽 테이블을 모두 합치는 `JOIN`입니다.        
JOIN하는 값을 기준으로 한 테이블에 해당 값이 없어도 NULL로 합치게 됩니다.

```SQL
SELECT *
FROM A
LEFT JOIN B ON A.id = B.id

UNION

SELECt *
FROM A
RIGHT JOIN B ON A.id = B.id;
```

다만 MySQL에서는 `FULL JOIN`을 지원하지 않기 떄문에 `UNION` 연산으로 테이블을 통합하여 처리해야 합니다.

이런 `FULL JOIN` 연산은 두 테이블 간에 빠진 데이터 혹은 잘못된 데이터를 찾을 때 유용하게 사용됩니다.        
혹은 데이터 분석 시 여러 테이블을 한 번에 처리하고자 할 때 사용할 수 있습니다.

## LEFT Join
왼쪽 테이블을 기준으로 데이터를 가져오는 `JOIN`입니다. 
왼쪽 테이블의 모든 데이터에 대한 정보를 볼 수 있고, `JOIN`하고자 하는 테이블에서 데이터가 없다면 `NULL`로 추가됩니다.

Hotel 테이블과 여기서 사용되는 image를 관리하는 Image 테이블을 관리하는 경우 `LEFT JOIN`을 사용할 수 있습니다.      
호텔은 반드시 존재하지만 이미지의 경우 존재하지 않을 수도 있기 때문에 `LEFT JOIN`을 사용하여 관리할 수 있습니다.

## RIGHT Join
이는 오른쪽 테이블을 기준으로 데이터를 가져오는 `JOIN`입니다.       
오른쪽 테이블의 모든 데이터에 대한 정보를 볼 수 있지만, `JOIN`하고자 하는 테이블에서 데이터가 없다면 `NULL`로 추가됩니다.

`RIGHT JOIN`의 경우, 모든 상품에 대한 데이터를 불러올 때 매출 정보를 보여주려고 한다는 예시를 들어보겠습니다.       
이 때 상품 중에서도 매출 정보가 없는 것도 불러와서 보여주고자 할 때 사용할 수 있습니다.
