# Pixel (px)
화소라고도 하며, 디지털 이미지를 나타내는 데 사용되는 최소 단위  
인치당 픽셀 수가 높을수록 해상도가 높다고 할 수 있다.   
ppi(pixel per inch) = dpi(dot per inch)  

안드로이드에서 애플리케이션에 이미지를 넣을 때, 픽셀 단위로 이미지를 넣으면 화면에 보이는 이미지 크기에 일관성이 없다 (pixel로 처리한 이미지는 핸드폰의 기종마다 다르게 보일 수 있다)  

# Dp (density-independent pixels, dip)
밀도 독립적인 픽셀을 나타내며, 이는 해상도와 관계 없이 이미지를 같은 비율로 표현한다는 의미가 된다.

160dpi를 갖는 화면에서 하나의 픽셀과 대응한다. 160dpi는 보통 320x480 크기의 해상도를 가지는 3.6인치 스크린을 말하는데, 이는 인치당 160개의 픽셀을 가지고 있다는 뜻
  
해상도에 따른 dp와 pixel 관계를 구하는 공식  
```
- dp = px * (160dpi / 단말dpi)
- px = dp * (단말dpi / 160dpi)
```

ex) 만약 각 이미지를 "3dp" size로 지정하게 된다면?  
단말 160dpi -> (1dp = 1pixel) 가로,세로 3 pixel  
단말 640dpi -> (1dp = 4pixel) 가로,세로 12pixel  
하지만 각각 dp로 지정했으므로 이미지 비율은 동일하게 처리된다.  

DP를 사용하는 것으로 해상도가 달라지긴 하지만 이미지의 비율을 유지할 수 있게 된다.  

이미지를 저장할 때 해상도 별 이미지를 별도의 resource 폴더로 구성하여 제공하는 것을 추천한다.  
ldpi / mdpi / hdpi / xhdpi / xxhdpi 로 각각 해상도별 이미지를 제공해주면 적합한 dpi에 맞는 이미지로 렌딩된다.

# SP (scale-independent pixels)
축적 독립적 픽세로 dp와 같이 밀도에 독립적인 픽셀이지만, 글꼴에 적용하기 위해 만들어진 단위를 말함  
UI 레이아웃 정의 시 텍스트 크기를 지정하기 위해 사용
  
안드로이드 시스템에서 시스템 텍스트 크기를 변경할 수 있는데, 안드로이드 모든 앱의 sp 처리된 부분은 영향을 받게 된다.
